  • Jmatcher: Job Matching Site

    Leading a Team in an Agile Environment | February 2017 - May 2017

    What It Is

    For my "Software Engineering 1" class, we had a single project that was 10 weeks long. We worked in teams of 4.

    We were assigned "Customers", Graduate Students that represented real life business customers. We came to them with software ideas, and they came up with a set of requirements for a software solution that we had to implement. These projects ranged from a cooking video website, to a fantasy soccer website, to a social media website.

    Our team developed a platform called "JMatcher", a Tinder-like job matching site. The idea is to match you to a job based the best fit. This was a 5 2-week sprint project where we had to implement many different features.

    What I Did

    My team, (and most of the class), constituted of Masters students. However, I ended up leading our team in the archetecture design and development of our sofware solution.

    Our framework of choice was Django (Python based web framework), mostly due to my prior experience at developing web apps. My rational was that it is "batteries included", and is excellent for rapid development and prototyping.

    I taught 3 other people Django in mini classes that I designed. I also led team meetings in all 5 sprints, and assigned development tasks at each sprint.

    Specific features I developed are Employer & Student User Accounts, Social Auth, Updating Profiles, Inbox, Live Chat, Heroku Deployments, and I also helped the out on Social Feed and Searching Jobs.

    Technology

    Backend Web Stack:

    Django - Python Web Framework

    PostgreSQL - Database

    Docker - Container for CI & Development

    Frontend Web Stack:

    Html + Bootstrap - Front End HTML Framework

    SCSS - Better CSS

    JQuery - Better JS

    Deployment & Monitoring:

    Heroku - Easy deployments

    Integrations:

    Mailgun - 3rd party email provider

    Oauth - Facebook/Google

    Why It Matters

    For the most part, this was agile software development where we met the customers every 2 weeks to evaluate progress and had daily scrums. In addition to the software requirements there were testing, documentation, and other project requirements. It was my first experience leading a team in software development.

    We ended up being the top project in the entire class. More impressively (and what I'm more proud of) is that we did this while spending (probably) the least amount of development time. We probably spent 7-10 hours / week each on development & project planning, but were extremely efficient!

  • Little Free Library: Research Project

    Research Assistant to Cognitive Science / Informatics Professor

    What It Is

    The idea is to provide a platform for “offline digital content”. A wireless signal is broadcast which users can connect to. Upon connection they are redirected to a page in their browser. At this page users can upload and share information anonymously. Everything exists locally, disconnected from the world wide web. That is, you have to be within the signal range to access any of the content.

    What I Did

    Created a piece of technology that could function as an isolated internet hub, a place where users can share information with one another anonymously.

    Sadly, I don't have a picture of what the final product looks like at the moment. I'm working on getting it. Imagine a raspberry pi connected to a portable Wifi hotspot device, both of which are connected to a portable USB charger. Hack away!

    Met with the Professor on a weekly basis to evaluate our goals.

    Technology

    Open WRT Custom Router Firmware - Hosting the website on the router directly!

    Django - Web backend, Python framework

    PostgreSQL - Database

    Apache - Server

    Why It Matters

    How do people interact when there are no reprocussions, no risk of being tracked or identified, where they can share anything they want unrestricted? There are many fun questions that can be asked and that can be studied from a project like this. I pose more scenarios on the project website.

    Show Me The Code!

    This is just a very short overview of what the project was. Please see the project website for more details:

  • Cerner Internship

    Continuous Integration / Continuous Deployment with Docker | June 2016 - August 2016

    What It Was

    An 8 week internship at Cerner, a medical software company providing solutions to private practices and hospitals alike.

    What I Did

    I Integrated a docker CI/CD solution into Jenkins deployments on DC/OS and Marathon, developing a python package to abstract and automate docker operations including building and testing, registry pushes, and marathon deployments.

    Well that was a lot. I used Docker to automate our CI/CD process for a few dozen Ruby on Rail's developers. Our "Population Health" project needed to be converted to be compatible with Docker - I worked on a small team to integrate this project with Docker and Apache Mesos (a container orchestration platform).

    Basically, I converted our web app to follow these principals: https://12factor.net/

    Why It Matters

    Dozens of Rails developers were able to save time working locally with a Docker set up. By setting up automating testing with Jenkins & Docker, I managed to streamline the CI process with our internal Docker Registry. The package I wrote helped write more maintainable and testable CI code.

    When I left, my team was in the process of setting up our CD process with Docker & DC/OS Apache Mesos. I helped kickstart our initial setup and managed to push our builds successfully to DC/OS Mesos.

    Technology

    Docker - Dev & Production containerization

    Ruby on Rails - Web Framework for our web app

    Python - Language I used to write a Docker engine wrapper

    Jenkins - Automating continuous integration and continuous deployment

    DC/OS, Marathon, & Apache Mesos - Container orchestration platform
